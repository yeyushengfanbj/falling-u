# falling-u 运营计划

开发路线图和京东联盟 API 对接计划见 `docs/DEVELOPMENT_PLAN.md`。

## 盈利模型

主要收入来自京东联盟 CPS：

```text
用户访问文章或工具页
-> 点击京东联盟推广链接
-> 跳转京东下单
-> 订单有效且未退款
-> 京东联盟按规则结算佣金
```

普通京东商品链接不会自动产生收入，需要在京东联盟后台生成带推广参数的链接。

可扩展收入：

- 京东联盟 CPS 佣金
- Google AdSense / 百度联盟广告
- 品牌商测评、专题页和广告位
- 邮件订阅商品推荐
- 自营软件、模板、课程或会员内容

## 开发与运营成本

| 阶段 | 一次性开发成本 | 月成本 | 目标 |
|---|---:|---:|---|
| MVP 内容站 | 0-3000 元 | 50-200 元 | 验证收录和点击 |
| 正式内容站 | 5000-20000 元 | 100-500 元 | 分类、搜索、SEO 模板、后台 |
| 自动化商品站 | 30000-100000+ 元 | 300-2000+ 元 | 商品库、价格更新、排行榜、API |

初期最贵的是内容，不是代码。建议先写 50-100 篇高质量内容，再决定是否做自动化商品库。

## 收入估算

收入公式：

```text
月佣金 = 月访问人数 × 点击率 × 下单转化率 × 客单价 × 佣金率
```

示例：

```text
10000 访问 × 8% 点击 × 2% 下单 × 300 元客单价 × 3% 佣金 = 144 元/月
```

现实区间：

| 阶段 | 月访问量 | 可能月收入 |
|---|---:|---:|
| 0-3 个月 | 很少 | 0-100 元 |
| 3-6 个月 | 几千到一两万 | 0-500 元 |
| 6-12 个月 | 1万-10万 | 100-3000 元 |
| 做得比较好 | 10万-50万 | 3000-20000 元 |
| 强 SEO 内容站 | 50万+ | 2万+ |

## 获客路线

不要推广网站名，先覆盖用户正在搜索的问题。

优先关键词类型：

- 千元显示器推荐
- Type-C 扩展坞推荐
- NAS 新手硬盘怎么选
- 家用 2.5G 交换机推荐
- 办公椅 京东 自营 推荐
- 宿舍小夜灯推荐

渠道：

- SEO：Google Search Console、百度搜索资源平台、站点地图。
- 知乎：长回答引流到完整清单。
- 小红书：预算清单、桌面搭建图文。
- B 站/抖音：选购避坑短视频。
- 社区论坛、导购群和生活经验帖：发布真正有用的清单或工具。

## 90 天执行计划

第 1-2 周：

- 上线站点基础页面。
- 建立 5 个核心分类：数码电器、电脑配件、存储网络、家居办公、生活小物。
- 商品库从首批 6 个验证商品扩到至少 30 个，每个分类至少 6 个。
- 写 10 篇有明确购买意图的指南。

第 3-6 周：

- 商品库扩到 50-80 个，优先补点击率高、搜索意图强的品类。
- 累计到 30 篇文章。
- 每篇文章做 3-5 个外部分发版本。
- 接入搜索站长平台和分析工具。

第 7-12 周：

- 根据点击数据淘汰低转化商品，保留有佣金和转化潜力的商品。
- 累计到 60-80 篇文章。
- 根据 Search Console 和百度搜索资源平台数据扩展长尾词。
- 找到点击率高的页面，优化商品卡片和推荐理由。

## 商品库标准

首批 6 个商品只用于验证网站结构，不是最终规模。商品库进入可运营状态前，至少要达到：

- 每个核心分类不少于 6 个商品。
- 全站不少于 30 个可购买商品。
- 每个商品有京东联盟短链，不使用普通京东商品链接作为购买入口。
- 商品图只使用可靠的京东商品图；短链落到活动页且无法确认图货一致时，先使用类型占位。
- 批量新增时先把京东分享文案放入 `imports/jd-shares.txt`，运行 `npm run import:jd` 生成草稿。
- 草稿复核后将 `importStatus` 改为 `ready`，运行 `npm run merge:drafts` 合并到正式商品库。
- 每批新增商品后运行 `npm run audit:catalog` 和 `npm run check:links`。

## 合规与部署

- 页面应披露“部分链接为联盟推广链接”。
- 不要承诺最低价、官方认证或未验证的商品参数。
- 中国大陆服务器需要 ICP 备案。
- 海外部署通常不需要备案，但国内速度和稳定性可能较弱。
